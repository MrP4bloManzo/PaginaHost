Protocolos de Red

Los puertos de red TCP (Protocolo de Control de Transmisión) y UDP (Protocolo de Datagramas de Usuario) son números de identificación que se utilizan en el nivel de transporte de un sistema de comunicación en red para dirigir el tráfico de datos entrante y saliente hacia aplicaciones y servicios específicos en una computadora o dispositivo en una red. Aquí hay una breve descripción de cada uno:

TCP (Protocolo de Control de Transmisión):

TCP es un protocolo de comunicación orientado a la conexión. Esto significa que antes de que se pueda enviar o recibir datos, se establece una conexión entre el emisor y el receptor. Los puertos TCP se utilizan para aplicaciones que requieren una comunicación fiable y ordenada de datos, como la transferencia de archivos (FTP), navegación web (HTTP), correo electrónico (SMTP, POP3, IMAP), acceso remoto (SSH), y muchas otras. Los números de puerto TCP van desde 0 hasta 65535.


UDP (Protocolo de Datagramas de Usuario):
UDP es un protocolo de comunicación sin conexión, lo que significa que no se establece una conexión previa antes de enviar datos. Esto lo hace más rápido pero menos confiable en comparación con TCP. Los puertos UDP se utilizan en aplicaciones donde la velocidad y la eficiencia son más importantes que la integridad de los datos, como la transmisión de video en tiempo real (por ejemplo, streaming), servicios de juegos en línea, servicios de voz sobre IP (VoIP) y DNS (Sistema de Nombres de Dominio). Al igual que con TCP, los números de puerto UDP van desde 0 hasta 65535.


HTTP
(Hypertext Transfer Protocol)

HTTP es el protocolo utilizado para la transferencia de datos en la World Wide Web. Funciona mediante solicitudes y respuestas, permitiendo que los clientes web (navegadores) soliciten recursos a servidores web (por ejemplo, páginas web) y reciban respuestas en forma de texto o recursos multimedia.
Puerto: 80

HTTPS
(Hypertext Transfer Protocol Secure)

HTTPS es una versión segura de HTTP que utiliza cifrado SSL/TLS para proteger la privacidad de la comunicación entre el cliente y el servidor web. Es esencial para garantizar la seguridad en transacciones en línea y para proteger la confidencialidad de los datos.
Puerto: 443

SMTP
(Simple Mail Transfer Protocol)

SMTP es el protocolo utilizado para enviar correo electrónico. Permite que los clientes de correo electrónico envíen mensajes a servidores de correo saliente para su entrega a destinatarios.
Puerto: 25

POP3
(Post Office Protocol 3)

POP3 es un protocolo utilizado por clientes de correo electrónico para recuperar mensajes de un servidor de correo. Los mensajes se descargan en el cliente y se eliminan del servidor (por defecto), lo que significa que los correos electrónicos se almacenan localmente.
Puerto: 110

IMAP
(Internet Message Access Protocol)

IMAP es otro protocolo de correo electrónico que permite a los clientes acceder y administrar correos electrónicos almacenados en un servidor de correo. A diferencia de POP3, los mensajes se mantienen en el servidor y pueden organizarse en carpetas, lo que permite el acceso desde múltiples dispositivos.
Puerto: 143

MIME
(Multipurpose Internet Mail Extensions)

MIME es una especificación que extiende SMTP para permitir el envío de contenido multimedia, como imágenes y archivos adjuntos, a través de correo electrónico. Define cómo se deben codificar y decodificar los datos adjuntos.
Puerto: No es un protocolo con puerto específico.

FTP
(File Transfer Protocol)

FTP es un protocolo utilizado para la transferencia de archivos entre un cliente y un servidor. Permite la carga y descarga de archivos de un servidor remoto.
Puerto: 21

FTPS
(FTP Secure)

FTPS es una extensión segura de FTP que utiliza SSL/TLS para cifrar la comunicación entre el cliente y el servidor FTP. Proporciona seguridad adicional a las transferencias de archivos.
Puerto: 990 (Control), 989 (Datos)

SFTP
(SSH File Transfer Protocol)

SFTP es un protocolo de transferencia de archivos seguro basado en SSH. Permite la transferencia segura de archivos y la administración de archivos en servidores remotos.
Puerto: 22

BitTorrent

BitTorrent es un protocolo de transferencia de archivos punto a punto (P2P) utilizado para compartir archivos grandes de manera eficiente. Los usuarios comparten fragmentos de archivos entre sí, lo que permite descargas más rápidas y distribución de carga.
Puerto: Varios (depende de la configuración)

eDonkey
(eDonkey2000)

eDonkey es un protocolo de red P2P utilizado para compartir archivos, similar a BitTorrent. Fue popularizado por programas como eMule y permite a los usuarios compartir archivos entre sí.
Puerto: 4662 (TCP), 4672 (UDP)

FastTrack

FastTrack es un protocolo de red P2P utilizado por programas como Kazaa y Grokster para compartir archivos. Fue popular en la década de 2000.
Puerto: No tiene un puerto estándar (depende de la aplicación)

IRC
(Internet Relay Chat)

IRC es un protocolo de chat en tiempo real que permite a los usuarios conectarse a servidores y participar en salas de chat. Se utiliza para la comunicación en línea en tiempo real.
Puerto: 6660-6669 (puertos no seguros)

XMPP
(Extensible Messaging and Presence Protocol)

XMPP es un protocolo de mensajería instantánea y presencia que se utiliza en servicios de chat en tiempo real y comunicación en línea. Es extensible y ampliamente utilizado en aplicaciones de mensajería.
Puerto: 5222 (Cliente), 5269 (Servidor)

MTProto

MTProto es el protocolo de cifrado utilizado por la aplicación de mensajería Telegram. Proporciona seguridad en las comunicaciones y es uno de los elementos clave para garantizar la privacidad en la aplicación.
Puerto: 443
